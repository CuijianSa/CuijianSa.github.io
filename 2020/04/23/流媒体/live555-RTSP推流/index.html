简介

RTSP主要流程

命令字

  • OPTION:客户端向服务器询问可用方法
  • DESCRIBE: 客户端向服务器请求媒体资源描述
  • SETUP: 确定具体的流媒体如何传输
  • PLAY: 请求播放媒体流
  • PAUSE: 暂停媒体流
  • TEARDOWN: 停止媒体流
  • GET_PARAMETER:检索URI数据中参数值
  • SET_PARAMETER: 设置媒体流的参数
  • REDIRECT：重定向请求，用于服务器通知客户端的新的服务地址
  • ANNOUNCE(RTSP2.0中被移除): 客户端向服务器端发布URL指定的媒体信息描述
  • RECORD(RTSP2.0中被移除): 请求录制指定范围的媒体数据，请求中可指定录制的起止时间戳

拉流流程

OPTION -> DESCRIBE -> SETUP -> PLAY

推流流程

RTSP不仅仅支持拉流，也可以支持推流

OPTION -> ANNOUNCE -> SETUP -> RECORD
